## Glossary

| Term | Definition |
|------|-----------|
| **Fund** | An accounting entity with its own restricted/unrestricted designation. Every transaction line is coded to exactly one fund. |
| **Restricted Net Assets** | Donor-restricted funds (FASB ASC 958: "Net Assets With Donor Restrictions"). Expenses against restricted funds trigger automatic net asset releases. |
| **Net Asset Release** | Automatic reclassification entry when restricted funds are spent: DR Net Assets With Donor Restrictions, CR Net Assets Without Donor Restrictions. |
| **Staging Table** | Database table owned by financial-system where internal apps INSERT approved financial data for processing. |
| **Trust-Escalation** | Matching model: suggest → user confirms → user creates rule → future matches auto-approved. |
| **Copilot** | Right-panel AI chatbot available on every page, with page-specific context and tools (D-129). |
| **CIP** | Construction in Progress — asset account accumulating development costs before building is placed in service. |

## 1. System-Wide Invariants

Non-negotiable rules enforced at database or application level. Violations are system bugs, not user errors.

| ID | Invariant | References |
|----|-----------|------------|
| INV-001 | **Double-entry balance.** For every transaction, Sum(debits) = Sum(credits). Rejected at write time if violated. | GL-P0-001 |
| INV-002 | **Valid account reference.** Every transaction line must reference an existing, active account_id. | GL-P0-002 |
| INV-003 | **Valid fund reference.** Every transaction line must reference an existing fund_id. | GL-P0-003, D-013 |
| INV-004 | **Active account constraint.** Inactive accounts cannot receive new transaction lines. History retained. | GL-P0-002 |
| INV-005 | **Fund restriction immutability.** A fund's restriction type (Restricted/Unrestricted) is set at creation and cannot change. To change: create new fund, transfer balances. | D-013 |
| INV-006 | **Transaction correction rules.** Unmatched posted transactions may be edited in place with full audit trail. Bank-matched transactions are immutable — corrections via reversal entry only. Voided transactions are excluded from GL totals but retained in audit trail. | D-053 |
| INV-007 | **Restricted fund auto-release.** When an expense posts to a restricted fund, the system atomically generates a net asset release entry (DR: Net Assets With Donor Restrictions, CR: Net Assets Without Donor Restrictions) for the same amount. No restricted-fund expense may exist without its paired release. | D-029 |
| INV-008 | **System-generated entry flag.** Entries generated by the system (depreciation, interest accrual, net asset releases, conversion adjustments) are flagged `is_system_generated = true` and are non-editable by users. Manual corrections require separate adjusting journal entries. | D-019, D-029 |
| INV-009 | **Ramp posting gate.** Uncategorized Ramp transactions are excluded from financial statements and cannot post to GL. Categorization (GL account + fund assignment) is a prerequisite for posting. | D-021 |
| INV-010 | **Fund-level trial balance integrity.** For any individual fund, Sum(debits) = Sum(credits). | D-013 |
| INV-011 | **Transaction source provenance.** Every transaction records `source_type` (MANUAL, TIMESHEET, EXPENSE_REPORT, RAMP, BANK_FEED, SYSTEM) and optional `source_reference_id`. Set at creation, immutable. | D-041 |
| INV-012 | **Immutable audit log.** Every GL write operation (transaction created, reversal posted, account created/deactivated, fund created/deactivated, reconciliation signed off, budget changed) recorded in append-only log with: timestamp, user, action type, affected entity ID, before/after state. Cannot be edited or deleted. | D-041 |
| INV-013 | **No deletions.** Soft delete only. All data retained permanently. GL accounts with history cannot be deleted, only deactivated. | Core |
| INV-014 | **No approval workflows.** All posted entries are final. No pending/approval states for financial transactions. Audit logging is the compensating control. | D-044 |
| INV-015 | **Accrual basis.** Revenue recognized when earned, expenses when incurred. Cash receipts/payments are balance sheet events, not P&L events. | D-005 |

### 9.3 GAAP Policies (Written Documentation)

These policies are organizational positions stored as system configuration, not features. They drive system behavior and are referenced in Notes to Financials (Report #20).

| Policy | Rule | Decision |
|--------|------|----------|
| Capitalization threshold | $2,500 per item/invoice (IRS de minimis safe harbor). Increases to $5,000 when audited financials required. | D-078 |
| Bad debt | Direct write-off method (expense when uncollectible). Mandatory annotation: reason, collection efforts, authorization. Switch to allowance method when audit threshold reached. | D-079 |
| Depreciation | Straight-line only. IRS standard useful lives. No accelerated methods, no Section 179, no bonus depreciation. Component for building, single-item for others. | D-080, D-127 |
| Construction interest | 100% of AHP interest capitalized to CIP - Construction Interest during construction period (ASC 835-20). Capitalization ends when all structures are placed in service. No pro-rata reduction on partial PIS — full capitalization continues until final structure converted. | D-022, DM-P0-026 |
| Functional allocation | Year-end per-GL-account percentage allocation to Program/Admin/Fundraising. Stored as metadata, not journal entries. | D-061 |
